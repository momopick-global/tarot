# 디자인 시스템 가이드

유어타로 서비스의 시각적 언어를 정의한 문서입니다. 개발자와 디자이너가 동일한 기준을 공유해, 웹부터 앱까지 일관된 브랜드 경험을 제공하는 것을 목표로 합니다. 아래 지침은 1차 오픈 기준이며 이후 업데이트에서 확장될 수 있습니다.

## 문서 목적

- **일관성 유지**: 색상, 타이포그래피, 구성 요소 사용 규칙을 명시하여 페이지 간 스타일 차이를 최소화합니다.
- **재사용성 확보**: 버튼, 카드, 모달 등 UI 컴포넌트를 정의하여 새로운 페이지 추가 시 중복 개발을 줄입니다.
- **확장성 보장**: 2차, 3차 업데이트(다국어, 결제, 앱화 등) 시 기존 시스템에 쉽게 새로운 디자인 요소를 추가할 수 있도록 준비합니다.
- **Figma 연동**: 모든 디자인 토큰(색상, 폰트, 레이아웃 값 등)은 Figma의 라이브러리와 연결하여 유지·관리합니다.

## 색상(Color Palette)

유어타로의 감성은 따뜻하고 신비로운 분위기를 기반으로 합니다. 기본 색상은 다음과 같습니다.

| 역할 | 색상 코드 | 설명 |
| --- | --- | --- |
| Primary | `#6F42C1` | 브랜드를 대표하는 보라색. 버튼, 하이라이트에 사용 |
| Primary Light | `#A992E2` | Primary 색상의 밝은 변형. 호버 상태나 배경에 사용 |
| Primary Dark | `#4E2B8C` | Primary 색상의 어두운 변형. 클릭 상태, 그림자에 사용 |
| Accent | `#F5C542` | 별빛처럼 따뜻한 금색. 포인트 요소나 아이콘에 사용 |
| Neutral 90 | `#202139` | 기본 텍스트 색. 검은색에 가까운 어두운 회색 |
| Neutral 60 | `#666666` | 보조 텍스트, 캡션에 사용 |
| Neutral 30 | `#BBBBBB` | 디바이더, 비활성 상태에 사용 |
| Neutral 10 | `#F7F7F7` | 카드 배경, 입력 필드 배경 |
| Error | `#E94444` | 오류 메시지, 위험 상태에 사용 |
| Success | `#28A745` | 성공 메시지, 완료 상태에 사용 |

### 색상 사용 원칙

- **접근성**: 텍스트와 배경 간 대비 비율은 WCAG AA 수준(최소 4.5:1)을 충족해야 합니다. Primary 색상 위에는 밝은 텍스트(예: 흰색)를 사용합니다.
- **일관성**: 버튼과 링크, 강조 텍스트에는 Primary 계열을 사용하며, 같은 요소에는 같은 색을 반복합니다.
- **강조와 절제**: Accent 색상은 사용자 행동을 유도하거나 중요 정보를 강조할 때만 사용합니다.
- **다크 모드 준비**: 1차 오픈에서는 밝은 테마를 기본으로 하지만, 색상 변수는 디자인 토큰으로 분리하여 다크 모드 테마 전환을 준비합니다.

## 타이포그래피(Typography)

유어타로는 감성적이면서도 읽기 쉬운 글꼴을 사용합니다. 기본 글꼴은 웹 표준 Sans-serif를 사용하되, Figma에서 지정한 한글/영문 폰트를 적용할 수 있습니다. 폰트 크기와 사용처를 정의합니다.

| 스타일 | 폰트 크기 | 글꼴 무게 | 행간 | 용도 |
| --- | --- | --- | --- | --- |
| H1 | 28px | 700 | 36px | 주요 제목(페이지 타이틀) |
| H2 | 24px | 600 | 32px | 섹션 제목 |
| H3 | 20px | 600 | 28px | 카드 제목, 결과 제목 |
| Body | 16px | 400 | 24px | 일반 본문 텍스트 |
| Caption | 14px | 400 | 20px | 설명, 보조 문구 |
| Button | 16px | 600 | 20px | 버튼 내 텍스트 |
| Label | 14px | 500 | 18px | 입력 폼 라벨 |

### 타이포그래피 가이드

- **가독성**: 긴 문장은 50~60자 길이를 넘지 않도록 하고, 문단은 3~5문장으로 나눕니다.
- **통일성**: 페이지 내 제목 계층 구조를 지키며, Heading과 Body 스타일을 혼용하지 않습니다.
- **다국어 지원**: 한국어를 기본으로 하되, 영문과 일본어를 고려하여 폰트 선택 시 합자 조합과 자간을 테스트합니다.
- **라인 높이(line-height)**: 충분한 공간을 줘서 텍스트가 숨쉬도록 합니다. 작은 폰트일수록 더 큰 행간을 사용합니다.

## 레이아웃 및 그리드(Layout & Grid)

- **모바일 퍼스트**: 디자인은 모바일 웹을 기준으로 시작합니다. 데스크톱/태블릿 화면에서는 요소를 재배치하되 동일한 UI 패턴을 유지합니다.
- **반응형 규칙**: 주요 브레이크포인트는 360px, 390px, 412px, 430px, 768px, 1024px입니다. 콘텐츠 폭은 1차 오픈에서 320~480px 사이를 최적화하고, 태블릿/데스크톱에서는 여백을 넉넉히 줍니다.
- **그리드 시스템**:
    - **모바일**: 4-컬럼 그리드(각 컬럼은 동일한 폭). 여백은 16px, 컬럼 간격은 8px을 기본으로 합니다.
    - **데스크톱**: 12-컬럼 그리드. 여백은 24px, 컬럼 간격은 16px을 기본으로 합니다.
- **간격(Spacing)**: 4px을 기본 단위로 사용하고, 8px, 16px, 24px 등 배수로 여백을 설정합니다. 카드와 카드 간 간격, 섹션 간 간격도 이 규칙을 따릅니다.
- **정렬**: 요소는 기본적으로 왼쪽 정렬을 사용하지만, 카드 목록 등 반복 요소는 가운데 정렬하여 시각적 균형을 맞춥니다.

## UI 구성요소(Component Library)

각 구성요소는 Figma 라이브러리로 관리하며, React 컴포넌트로 구현됩니다. 컴포넌트는 재사용성을 위해 Atomic Design 패턴(Atoms, Molecules, Organisms)을 따릅니다. 아래는 1차 오픈에서 필요한 주요 요소와 사용 규칙입니다.

### 버튼 (Buttons)

- **Primary Button**: 브랜드 컬러(Primary) 배경, 흰색 텍스트. 사용자 주요 행동(카드 뽑기, 결과 보기)에 사용합니다.
- **Secondary Button**: 중간 밝기의 Neutral 10 배경, Primary 색 텍스트. 보조 행동(다시 뽑기, 마스터 선택) 등에 사용합니다.
- **Ghost Button**: 배경 없이 텍스트만 강조하거나 경계선만 있는 버튼. 취소나 링크 스타일 버튼에 사용합니다.
- **상태**: 기본, 호버, 눌림, 비활성 상태를 정의합니다. 호버 시 Primary Light 색상으로, 눌림 시 Primary Dark 색상으로 전환합니다. 비활성 상태는 Neutral 30 색을 사용합니다.

### 카드(Card Components)

- **타로 카드 표시**: 78장의 카드를 표준 크기(비율 3:5)로 표현합니다. 이미지 위에는 아무 텍스트도 없으며, 카드 이름은 이미지 아래에 작은 본문 스타일로 별도 표기합니다.
- **마스터 카드**: 마스터 목록에서 사용할 프로필 카드입니다. 원형 아바타와 이름, 짧은 설명(한 줄)을 포함합니다. 클릭 시 별도 상세 페이지(`masters/[slug].tsx`)로 이동합니다.
- **결과 카드**: 결과 페이지에서 사랑/일/금전/조언 섹션을 각각 별도의 카드로 나눠 표시합니다. 각 카드에는 아이콘, 제목(H3), 내용(Body)이 포함됩니다.

### 입력 필드 및 폼(Form Elements)

- **TextField**: 라벨(Label), 입력 영역, 에러 메시지로 구성됩니다. 포커스 시 Primary 색상으로 밑줄이 활성화됩니다.
- **TextArea**: 의견/제휴 문의와 같이 긴 문구를 입력할 때 사용합니다. 기본 높이는 4줄이며, 포커스 상태와 에러 상태를 정의합니다.
- **Checkbox / Toggle**: 뉴스레터 구독 등 이진 선택이 필요한 경우 사용합니다. 기본 Primary 색상으로 체크 표시를 표현합니다.
- **reCAPTCHA Badge**: 의견/제휴/탈퇴 폼 하단에 작은 뱃지로 표시되며, UI 간섭을 최소화하도록 배치합니다.

### 네비게이션(Navigation)

- **상단 네비**: 로고와 햄버거 메뉴 또는 탭 메뉴를 포함합니다. 작은 화면에서는 아이콘 메뉴를 사용하고, 큰 화면에서는 텍스트 탭을 사용합니다.
- **하단 네비(모바일)**: 자주 사용하는 페이지(홈, 마스터, 오늘의 타로, 마이페이지)를 아이콘과 함께 하단에 고정할 수 있습니다. 공간이 부족하면 상단 네비게이션 메뉴에 통합합니다.

### 모달 및 오버레이(Modal & Overlay)

- **카드 뽑기 애니메이션**: 사용자가 카드를 선택할 때 전체 화면을 덮는 오버레이를 사용하여 집중도를 높입니다. 애니메이션은 0.5초 이내로 실행되고, 완료 후 결과 페이지로 이동합니다.
- **알림 모달**: 탈퇴 확인, 결과 저장 확인 등 사용자에게 중요한 결정을 요청할 때 사용합니다. 모달 안에는 제목, 메시지, Primary/Secondary 버튼이 포함됩니다.

### 아이콘(Iconography)

- **Font Awesome** 및 **lucide-react** 아이콘 라이브러리를 사용합니다. 선(line) 형태의 아이콘을 기본으로 하여 미니멀한 느낌을 줍니다.
- 아이콘 크기는 버튼 내에서는 20px, 독립 사용 시에는 24px, 카드 섹션에서는 16px으로 통일합니다.
- 의미 전달을 위해 아이콘 옆에는 텍스트를 함께 표시합니다. 단독 아이콘 사용은 최소화합니다.

## 모션 및 애니메이션(Motion)

유어타로의 상호작용은 부드럽고 자연스러워야 합니다. 과도한 애니메이션은 피하고, 사용자의 행동에 즉각적인 피드백을 제공합니다.

- **카드 뒤집기**: 카드 선택 시 Y축을 중심으로 180도 회전하는 애니메이션을 사용합니다. duration은 0.6초 이내로 설정합니다.
- **버튼 호버 효과**: 색상 변화와 함께 그림자(Shadow) 효과를 주어 클릭 가능성을 강조합니다.
- **스켈레톤(Skeleton)**: 데이터 로딩 중에는 반투명한 스켈레톤 컴포넌트로 레이아웃 형태를 보여줍니다. 페이드 인/아웃 효과를 주어 자연스럽게 전환합니다.

## 이미지(Image Guidelines)

- **타로 카드 이미지**: 해상도는 최소 600×1000px 이상이며, PNG 또는 WebP 형식을 사용합니다. 78장의 카드 이미지는 동일한 스타일로 디자인되어야 하고, Figma 라이브러리에서 관리합니다.
- **마스터 프로필**: 정사각형(1:1) 비율의 이미지 파일을 사용합니다. 배경은 투명 또는 브랜드 컬러 계열로 통일합니다.
- **OG 이미지**: 공유용 Open Graph 이미지는 1200×630px 비율로 준비합니다. 결과 요약과 카드 이미지, 마스터 이름을 포함하며, SEO/공유 문서에서 자세히 다룹니다.
- **압축 및 최적화**: 모든 이미지 파일은 빌드 시점에 `sharp` 또는 비슷한 도구로 압축하고, 브라우저에서는 적절한 포맷(WebP/AVIF)으로 제공하여 페이지 로딩 시간을 줄입니다.

### 덱 스타일(Deck Style)

유어타로의 9명 마스터는 각기 다른 카드 세트를 사용합니다. 마스터의 캐릭터에 맞춰 덱 스타일을 커스터마이징하지만, 기본 레이아웃과 비율은 일관되게 유지해야 합니다.

- **비율 유지**: 모든 덱의 카드는 3:5 비율과 최소 600×1000px 해상도를 유지합니다. 이는 결과 페이지와 카드를 선택하는 화면에서 동일한 크기로 렌더링할 수 있게 합니다.
- **스타일 가이드**: 각 덱은 마스터의 성격을 반영한 색상과 일러스트를 사용하되, 지나치게 이질적인 디자인은 피합니다. 예를 들어 몽환적인 마스터는 파스텔 톤과 흐린 배경을, 논리적인 마스터는 명확한 라인과 절제된 색상 조합을 사용할 수 있습니다.
- **색상 테마**: 덱의 주 색상과 보조 색상은 `card_decks.primary_color`와 `card_decks.accent_color`에 정의합니다. 디자인 시스템의 컬러 팔레트와 조화를 이루도록 합니다.
- **파일 구조**: 각 덱의 이미지는 `/public/images/cards/{deck_slug}/{card_id}.png` 또는 `.webp` 경로에 저장합니다. `{deck_slug}`는 마스터 slug와 동일하게 설정하여 디렉터리 구조가 명확해집니다. 예: `public/images/cards/cassian/00.png`.
- **버전 관리**: 덱 이미지가 업데이트될 경우 파일 이름을 교체하지 않고 버전 폴더(`v1`, `v2`)를 생성하여 관리하면 캐시 문제를 줄일 수 있습니다.

## 접근성(Accessibility)

- **키보드 네비게이션**: 모든 버튼과 입력 필드는 키보드 포커스 순서에 포함되어야 합니다. 포커스 시 테두리와 색상 변화를 통해 명확하게 표시합니다.
- **스크린 리더**: 이미지에는 의미 있는 `alt` 텍스트를 제공하고, 아이콘에는 `aria-label`을 설정하여 스크린 리더 사용자를 지원합니다.
- **색상 대비**: 앞서 언급한 색상 대비 기준을 따릅니다. Error/Success 메시지는 텍스트와 아이콘 모두 명확히 구분되도록 색상을 조정합니다.
- **애니메이션 감소**: 사용자 설정(예: prefers-reduced-motion)을 감지하여 애니메이션을 최소화합니다.

## Figma 사용 가이드

- **컴포넌트 정의**: 모든 버튼, 카드, 아이콘, 모달 등은 Figma에서 컴포넌트로 정의하고, Variants(상태, 크기, 색상)를 설정합니다.
- **토큰 관리**: 색상, 폰트 크기, 여백 값은 디자인 토큰 플러그인(Figma Tokens)을 활용하여 변수로 관리합니다. 이렇게 하면 다크 모드나 리브랜딩 시 전체 테마를 쉽게 변경할 수 있습니다.
- **레이어 명명 규칙**: `component/variant/state` 형식으로 레이어 이름을 지정합니다. 예: `Button/Primary/Hover`, `Card/Tarot/Base`.
- **버전 관리**: 주요 변경 사항이 있을 때마다 Figma 버전 히스토리를 남기고, 개발자는 변경 내역을 확인 후 코드에 반영합니다.

## 향후 확장

- **다크 모드**: 디자인 토큰을 이용해 Primary, Neutral 색상을 어둡게 변환한 버전을 정의해 두면 추후 다크 모드 구현이 수월합니다.
- **애니메이션 라이브러리**: Framer Motion 등 React 애니메이션 라이브러리를 도입하여 복잡한 상호작용을 쉽게 구현할 수 있습니다.
- **추가 언어 지원**: 영문/일문 디자인 시 글자 길이와 줄바꿈 규칙이 달라지므로, 레이아웃에서 텍스트 영역의 최소/최대 폭을 고려합니다.
- **콘텐츠 유형 확장**: 연애타로, MBTI 테스트 등 다른 콘텐츠를 추가할 경우, 결과 카드 구조를 확장하거나 새 카테고리 컴포넌트를 정의합니다.
- **앱 전환**: 디자인 시스템이 React Native로도 재사용될 수 있도록, 기본 단위와 색상, 아이콘 세트를 독립 패키지로 유지합니다.
